对比一

CPU 技术路线

采购参数写"ARMv8 或 x86_64 + 安全可靠目录"——前三条路线均可投标保持竞争,龙芯被参数自然排除(有电信集采、中信银行招标同款措辞先例)。

★ 最推荐

海光 7000 系x86_64

★★★★★

国产 x86,与现有开发环境同架构

  • 与开发环境同架构,迁移零交叉架构风险
  • Node / Prisma / sharp 全部官方一级支持
  • 整机选择多:中科可控 H620、新华三 R4930、联想开天 KR722h
  • 性价比高,渠道充足
  • 源自 AMD Zen 授权,"自主度"分档低于鲲鹏/飞腾(企业内部采购无碍)
双路整机约 2 ~ 6 万(2×7255/7285,64~256GB)

鲲鹏 920ARM64

★★★★

信创供货面最强的 ARM 路线

  • 电信集采 ARM 标包占 70.7%,供货面最强
  • 整机厂多:华鲲振宇、宝德、长江计算、昆仑技术等
  • Node / Prisma / sharp 均有官方 linux-arm64 组件
  • 与开发环境(x86)不同架构,需一轮完整 arm64 适配验证
  • 7nm 先进制程受限,920 代际更新慢
双路 2.2 ~ 8.5 万(按 32/48/64 核档位)

飞腾 S5000CARM64

★★★

自主度评级最高档的 ARM 路线

  • FTC862 全自研核,自主度评级高
  • 党政市场认可度高
  • 软件兼容路径与鲲鹏一致(ARMv8)
  • 单核性能弱于海光 / 鲲鹏
  • 整机与渠道少
  • 上代 S2500 仍在售,需甄别
双路整机询价(渠道较少)
排除

龙芯 3C/3DLoongArch

—

全自研指令集路线

  • 指令集自研,自主可控叙事最强
  • Node.js 仅实验级支持、无官方二进制,社区版滞后数月
  • Prisma / sharp 无 loongarch 产物,需自行编译长期维护
  • 安全补丁跟进风险高 → 硬排除
不适用
结论:海光与鲲鹏均满足采购参数,评标时优先;首选海光(与开发环境同架构、交付风险最低),鲲鹏为强备选。
对比二

国产数据库

采购参数一条线锁定:"基于 PostgreSQL 内核 ≥9.6 + 标准 PG 驱动直连 + md5/scram 认证"——前两家够格竞争,后两家被参数排除;适配验证写为中标方义务 + 付款前置。

★ 最推荐

瀚高数据库(安全版)

★★★★★

PostgreSQL 内核直系(V5 与 PG10 同步)

  • 协议 / SQL 全兼容,标准 PG 驱动直连风险最低
  • 信创目录内
  • PG 生态文档与运维经验全部适用
  • 市场声量低于金仓 / 达梦
  • Prisma 无官方认证(靠合同适配条款 + 验收兜底)
按 CPU 授权,询价(预算 5~15 万含测试授权)

人大金仓 KingbaseES

★★★★

市占前列的 PG 系,PG 兼容模式

  • 案例最多、服务网点广
  • PG 兼容模式有 node-postgres 直连实测案例
  • 资质齐全
  • 需指定 PG 兼容模式(DB_MODE=pg)
  • version() 返回串不含 "PostgreSQL",驱动探测有坑
  • Prisma 无公开成功案例
同量级,询价
不满足参数

openGauss / GaussDB 系

—

华为生态,源自 PG 9.2 深度改造

  • 性能优化强、社区活跃
  • 华为体系整合好
  • 内核 PG 9.2.4,低于应用 ORM 最低支持线 9.6
  • 默认 sha256 认证使标准 PG 驱动连不上(需改 md5 并重置密码)
开源版免费 / 商业版询价
排除

达梦 DM8

—

党政市占第一的自研库

  • 名录资质最全
  • 官方 Node 驱动 dmdb 存在且持续更新
  • 无 Prisma 支持,官方 ORM 仅 TypeORM / Knex
  • 选它 = 数据访问层重写,预计数月工期 → 硬排除
询价
结论:瀚高最优先(内核直系),金仓为合格竞争者,两家保证竞争充分;openGauss 与达梦被同一条参数合规排除。
对比三

服务器操作系统

两者对本系统完全等效(glibc≥2.28 满足 Node 官方二进制最低线),双产品并列写入参数保持竞争。

★ 最推荐

银河麒麟 V10 SP3+

★★★★★

服务器信创市占第一

  • 与各整机互认证证书覆盖最全
  • glibc 2.28 满足 Node 官方二进制要求
  • 鲲鹏 / 飞腾 / 海光等六大平台同产品线
  • 授权约 0.5~1 万/台(含 3 年服务)
  • 自带软件源 Node 版本旧(实际部署用官方二进制,无碍)
约 0.5 ~ 1 万 / 台(永久授权 + 3 年服务)

统信 UOS 服务器版 V20

★★★★

Debian 系,生态亲和的等效备选

  • glibc 2.28 同样达标
  • 鲲鹏 920 / 海光 7000 全支持
  • 价格相近,可有效压价
  • 服务器端市占与互认证证书数量少于麒麟
同量级,询价
结论:评标按整机互认证完整度取舍;无论中哪家,系统部署方式零差别。
对比四

国密传输方案

本系统为企业内部系统、尚未等保定级;等保三级也不强制硬件密码机(软件密码模块二级可达标)——先零成本软件方案,定级后按密评结论补硬件。

★ 最推荐

开源国密网关 + 自建 CA

★★★★★

零采购成本的主方案

  • 0 元:Tengine + 铜锁(开放原子基金会项目,阿里云生产验证)
  • SM2 + RSA 双证书,非国密浏览器自动回退
  • 内网终端推根一次配置、长期有效
  • 需自行部署维护
  • 无对外公信力(纯内网够用)
0 元(实施计入系统部署)

公信 CA SM2 证书

★★★

对外可信场景的补充项

  • 对外 / 对第三方可信
  • CFCA、沃通等签发渠道成熟
  • 年费持续支出
  • 内网场景非必需
OV 型约 0.4 ~ 1 万 / 年

硬件密码机 + 签名验签

★★

密评最稳妥路径,按触发条件启动

  • 密钥硬件级保护,密码模块等级更高
  • 密评通过最稳妥
  • 投入 25~34 万
  • 未定级三级前属超前投入
密码机 8~12 万/台 ×2 + 签名验签 8~10 万
结论:主方案零成本即实现全站加密访问;第 9 项可选包仅在等保三级定级 / 密评启动时采购。
对比五

AI 能力供给

以"内网出网政策"为触发条件分批决策——允许出网则维持现状零投入;禁止出网时入门档即可覆盖现有全部 AI 功能。

★ 最推荐

维持云端 API(现状)

★★★★★

零新增投入,一切已跑通

  • 0 新增投入,全部 AI 功能已在用(DeepSeek / 豆包 / 千问)
  • 模型升级零成本
  • 系统支持按功能降级,随时可切私有化
  • 依赖内网出网通道
  • 业务数据出内网需合规评估
按量付费(现有支出不变)

昇腾入门档

★★★★

禁止出网时的首选私有化方案

  • 信创合规:Atlas 300I Duo ×2~4 或蒸馏版一体机
  • DeepSeek-R1-32B 量化 + Qwen2.5-VL-7B,覆盖现有全部 AI 功能
  • MindIE 生态成熟(80+ 企业适配)
  • 并发有限(几十级)
  • 新增昇腾运维技能
  • 图像生成类(AI 头像 / 3D 生成)暂无好的私有化平替
15 ~ 30 万一次性

昇腾标准档

★★★

满血模型、百人并发的升级档

  • 满血 671B 模型、百人并发
  • 一步到位(Atlas 800I A2,8×910B)
  • 远超当前实际负载
  • 应先用入门档验证真实用量再升级
80 ~ 150 万一次性
不推荐

NVIDIA L20 / 4090

—

非信创的性价比路线

  • 生态最成熟、单位算力最便宜(4090 约 1.2~1.5 万/卡)
  • 不在信创目录
  • 受美出口管制,渠道合规存疑
  • 4090 为消费卡,无企业保修
10 ~ 30 万(合规风险自担)
结论:AI 算力不进基础包,以出网政策为触发条件分批决策,避免为不确定需求提前锁定大额投入。
