# 党建益友 · 企业应用底座建设规划

> 版本：v1.0  
> 定位：以微内核 + 插件化架构构建的企业级党建工具箱平台,通过"搭积木"方式持续扩展业务能力,统一身份/权限/数据/UI,支持作为统一登录中心或接入集团认证体系。

---

## 一、总体设计理念

### 1.1 核心原则

| 原则 | 说明 |
|---|---|
| **微内核 + 插件化** | 平台只做底座,业务能力以插件形式装配 |
| **一次认证,处处可用** | 内置 IdP,可统一登录群晖、SeaTable 等内网系统 |
| **可对接集团认证** | 作为 SP 接入集团统一认证 (OIDC/SAML/CAS/LDAP) |
| **党组织原生建模** | 组织树天然支持党委/总支/支部/党小组层级 |
| **信创友好** | 国产化适配 (麒麟/统信/达梦/人大金仓/瀚高) |
| **设计统一** | 党建红 + 金黄 + 白底,全平台一致视觉语言 |

## 二、技术栈选型

### 2.1 推荐技术栈 (信创可替换)

| 层级 | 主选 | 信创替代 |
|---|---|---|
| 后端框架 | Spring Boot 3.x + Spring Security | 同 (JDK 17+,适配毕昇/华为 JDK) |
| ORM | MyBatis Plus / JPA | 同 |
| 前端框架 | Vue 3 + TypeScript + Vite | 同 |
| 微前端 | qiankun / Module Federation | 同 |
| UI 组件 | Element Plus + 党建主题定制 | 同 |
| 数据库 | PostgreSQL 15 / MySQL 8 | 达梦 DM8 / 人大金仓 / 瀚高 |
| 缓存 | Redis 7 | Redis (信创版) / Tair |
| 消息队列 | RabbitMQ | RocketMQ |
| 文件存储 | MinIO | MinIO (信创) / OBS |
| 网关 | Spring Cloud Gateway | 同 |
| 容器 | Docker + K8s | KylinDB OS + 信创 K8s |
| 监控 | Prometheus + Grafana + Loki | 同 |

### 2.2 为何选这套

- **后端 Java**:党政场景主流,信创生态成熟,招聘容易
- **前端 Vue 3**:相比 React 学习曲线友好,Element Plus 易于党建主题定制
- **qiankun**:沙箱隔离,插件独立开发部署,故障互不影响
- **PostgreSQL 优先**:schema 隔离能力强,适合插件数据隔离

## 三、插件规范 (核心)

### 3.1 插件清单文件 `plugin.yaml`

每个插件根目录必须包含此文件,平台据此安装、注册、鉴权。

```yaml
apiVersion: dyy.plugin/v1
kind: Plugin

metadata:
  # 全局唯一标识,小写字母+连字符
  name: certificate-manager
  version: 1.0.0
  displayName: 证书管理系统
  description: 提供党员证书、培训证书的签发、管理、验真能力
  author: 党建益友官方
  category: business       # business | tool | integration | theme
  icon: assets/icon.svg
  homepage: https://...
  license: Apache-2.0

spec:
  # 平台版本约束
  platform:
    minVersion: 1.0.0
    maxVersion: 2.x

  # 依赖其他插件
  dependencies:
    - name: file-storage
      version: ">=1.0.0"

  # 申请的权限 (安装时用户需确认)
  permissions:
    - user:read              # 读取用户基本信息
    - org:read               # 读取组织树
    - file:write             # 写文件
    - notification:send      # 发送站内通知

  # 前端路由注册
  routes:
    - path: /plugins/certificate
      component: pages/Index
      meta:
        title: 证书管理
        requireAuth: true

  # 菜单注入
  menu:
    - title: 证书管理
      icon: solar:diploma-bold
      order: 100
      path: /plugins/certificate
      permission: certificate:view
      children:
        - title: 我的证书
          path: /plugins/certificate/mine
        - title: 证书签发
          path: /plugins/certificate/issue
          permission: certificate:issue

  # 后端 API 前缀
  api:
    prefix: /api/plugins/certificate
    healthCheck: /health

  # 事件订阅与发布 (事件总线)
  events:
    publish:
      - certificate.issued
      - certificate.revoked
    subscribe:
      - user.created
      - org.member.transferred

  # 数据资源隔离
  resources:
    database:
      schema: plugin_certificate    # 独立 schema
      migrationPath: db/migration
    storage:
      bucket: plugin-certificate    # 独立 bucket
    cache:
      namespace: cert:              # Redis key 前缀

  # 配置项 (会在管理后台生成配置界面)
  config:
    - key: defaultIssuer
      label: 默认签发机构
      type: string
      required: true
    - key: certTemplate
      label: 证书模板
      type: file
      accept: .pdf,.png

  # 定时任务
  cron:
    - name: expireCheck
      schedule: "0 0 2 * * ?"
      handler: ExpireCheckJob

  # 提供的扩展点 (供其他插件调用)
  extensionPoints:
    - id: certificate.verify
      description: 提供证书验真服务
```

### 3.2 插件生命周期

```
┌──────────┐   install    ┌──────────┐   enable   ┌──────────┐
│  上传    │ ───────────▶ │  已安装   │ ─────────▶ │  运行中   │
└──────────┘              └──────────┘            └────┬─────┘
                               ▲                       │
                          uninstall                disable
                               │                       ▼
                          ┌──────────┐ ◀────────  ┌──────────┐
                          │  已卸载   │            │  已停用   │
                          └──────────┘            └──────────┘
                                                       │
                                                    upgrade
                                                       ▼
                                                  (回到运行中)
```

每个阶段触发对应钩子:`onInstall`、`onEnable`、`onUpgrade`、`onDisable`、`onUninstall`。  
卸载时**必须支持**:数据导出、Schema 清理、菜单/路由摘除、缓存清理。

### 3.3 插件 SDK API (供插件调用)

```typescript
// 后端 SDK (Java)
@Autowired private PlatformContext ctx;

ctx.user().current();                    // 当前登录用户
ctx.user().findById(id);
ctx.org().getTree();                     // 组织树
ctx.org().getPartyBranch(userId);        // 获取所在党支部
ctx.permission().check("certificate:issue");
ctx.event().publish("certificate.issued", payload);
ctx.event().subscribe("user.created", handler);
ctx.storage().put(bucket, key, stream);
ctx.notification().send(userId, msg);
ctx.audit().log(action, target, detail);
ctx.config().get("defaultIssuer");
ctx.i18n().t("cert.title");
```

```typescript
// 前端 SDK (TypeScript)
import { usePlatform } from '@dyy/sdk';

const { user, org, permission, event, http, i18n, theme } = usePlatform();

user.current.value;                      // 响应式当前用户
permission.has('certificate:issue');
event.emit('certificate.issued', data);
event.on('user.created', handler);
http.get('/api/plugins/certificate/list'); // 自动带 token
```

### 3.4 插件隔离机制

| 维度 | 隔离方式 |
|---|---|
| **数据库** | 独立 schema (PostgreSQL) 或独立库前缀 (MySQL) |
| **缓存** | Redis key 命名空间前缀 |
| **存储** | 独立 bucket |
| **前端** | qiankun JS 沙箱 + CSS 作用域 |
| **接口** | 网关按 `/api/plugins/{name}/*` 路由,权限校验前置 |
| **依赖** | 插件依赖打包到自身,避免版本冲突 |
| **日志** | 按插件名打 tag,独立采集 |

## 五、权限与认证体系

### 5.1 数据模型

```
User (用户)
  ├─ 多对多 → Role (角色)
  │            └─ 多对多 → Permission (权限点)
  ├─ 多对多 → Organization (组织)
  │            └─ 树形结构,带类型 (党委/总支/支部/小组/部门)
  └─ 多对一 → PartyIdentity (党员身份信息)
              ├─ 政治面貌 (党员/预备党员/积极分子/群众...)
              ├─ 入党时间
              ├─ 所在支部
              └─ 党内职务
```

### 5.2 权限粒度 (四级)

| 级别 | 说明 | 示例 |
|---|---|---|
| 菜单权限 | 控制菜单是否可见 | `certificate:menu` |
| 操作权限 | 控制按钮/接口是否可用 | `certificate:issue` |
| 数据权限 | 行级,本人/本支部/本党委/全部 | `scope: branch` |
| 字段权限 | 列级,敏感字段脱敏 | 身份证号、电话 |

### 5.3 党组织层级建模

```
党委
 ├── 党总支
 │    ├── 党支部
 │    │    └── 党小组
 │    │         └── 党员
 │    └── 党支部
 └── 党总支
```

每级组织自带:
- 书记 / 副书记 / 委员 角色
- 默认数据可见范围
- 上下级关系自动继承

### 5.4 作为 IdP (统一登录中心)

党建益友对外提供标准协议端点,可作为群晖、SeaTable 等系统的登录后端:

| 协议 | 端点 | 适用对接 |
|---|---|---|
| **OIDC** | `/oauth2/authorize`、`/oauth2/token`、`/oauth2/userinfo`、`/.well-known/openid-configuration` | SeaTable、现代 Web 应用 |
| **OAuth 2.0** | 同上 | API 授权 |
| **SAML 2.0** | `/saml2/metadata`、`/saml2/sso`、`/saml2/slo` | 群晖 DSM (企业版)、传统系统 |
| **CAS 3.0** | `/cas/login`、`/cas/serviceValidate` | 校园、政企老系统 |
| **LDAP** | `ldap://host:389`、`ldaps://host:636` | 群晖 DSM、邮件服务器、Linux 主机 |

**典型对接示例**:

- **群晖 DSM**:控制面板 → 域/LDAP → 加入 LDAP 服务器,填写党建益友 LDAP 地址即可
- **SeaTable**:管理员后台 → SSO 设置 → 启用 OIDC,填写 Client ID/Secret 和 Issuer URL
- **GitLab/Jenkins/Confluence**:OIDC 或 SAML
- **企业微信/钉钉**:反向作为身份源同步

### 5.5 作为 SP (接入集团统一认证)

提供**协议适配器**,集团认证侧只要支持其一即可接入:

```yaml
# 配置示例:接入集团 OIDC
auth:
  provider: external-oidc
  oidc:
    issuer: https://idp.group.com
    clientId: dangjianyiyou
    clientSecret: ${OIDC_SECRET}
    scopes: [openid, profile, email, groups]
    userMapping:
      username: preferred_username
      name: name
      email: email
      partyBranch: ext_party_branch    # 自定义字段
    autoCreate: true                    # JIT provisioning
    syncGroups: true
```

支持的对接方式:
- **集团有 OIDC/OAuth2** → 直接对接
- **集团是 SAML 2.0** → SAML 适配器
- **集团是 CAS** → CAS Client
- **集团是 LDAP/AD** → LDAP 同步 + LDAP Bind 登录
- **集团是自研协议** → 实现 `AuthProvider` 接口,开发适配器

### 5.6 单点登录与单点登出 (SSO/SLO)

- 用户在党建益友登录后,跳转 SeaTable/群晖自动登录
- 在任一系统登出,全局会话同步失效 (基于 OIDC `end_session_endpoint` 或 SAML SLO)

## 七、安全与合规

### 7.1 安全要求

| 项 | 要求 |
|---|---|
| 等保 | 二级起步,目标三级 |
| 密码学 | 支持国密 SM2/SM3/SM4,密码存储 BCrypt + 加盐 |
| 传输 | 全站 HTTPS,内部 mTLS 可选 |
| 会话 | JWT + Refresh Token,支持踢人下线 |
| 越权 | 接口级 + 数据级双重校验 |
| 审计 | 全操作留痕,登录、敏感操作、权限变更必审计 |
| 验证码 | 登录失败次数限制 + 滑块验证 |
| 注入防御 | 参数化查询、CSP、XSS 过滤 |
| 文件上传 | 类型白名单 + 病毒扫描 + 隔离存储 |
| 插件签名 | 官方插件强制签名校验,第三方可配置策略 |

### 7.2 数据合规

- 个人信息处理符合《个人信息保护法》
- 敏感字段加密存储 (身份证、电话)
- 数据导出脱敏可配置
- 跨境传输关闭 (信创场景)
