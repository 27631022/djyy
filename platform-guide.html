如何管理导航

进入后台 → 左侧菜单「首页导航」→ 可对分类和导航项进行增删改排序。

  1. 先新建「分类」（如"业务系统"、"党务系统"），设置图标和排序
  2. 在分类下新建「导航项」，填写名称、URL、图标
  3. 图标从内置 Lucide 图标库中选择，输入关键词实时搜索
  4. 拖动排序条调整显示顺序，保存后前台即时生效
💡 图标选择技巧：图标选择器支持中英文搜索，如"建筑""party""shield"等。鼠标悬停在图标上可看到名称。
🏢 组织管理
管理行政机构树和党组织树，支持最多4级行政层级

两棵组织树

🏛️
行政机构
一级单位 → 二级单位 → 三级单位 → 四级单位，描述公司行政结构
🔴
党组织
党委 / 党总支 / 党支部 / 临时党支部 / 党小组，描述党务组织架构

新建/编辑组织

  1. 进入后台「组织管理」，默认显示「行政机构」tab（左侧）
  2. 点击某节点右侧的「⋯」菜单，选择「添加子级」或「编辑」
  3. 填写名称、类型、上级节点，可选填「部门负责人」
  4. 保存后树状结构立即更新

成员管理

编辑行政机构时，切换到「成员」tab 可管理该部门的直接成员。

  • 添加成员：搜索姓名或工号，选择用户，可同时指定在该部门的「职务」
  • 新建用户：若用户不存在，可在弹窗内直接新建后添加
  • 移出成员：点击成员行右侧「移出」，即时生效（不删除用户账号）
💡 主部门规则：用户加入第一个同类型组织时自动设为「主部门」。有多个归属时，删除主部门会自动将最早加入的其他部门提升为主部门。

部门负责人（重要）

部门负责人用于「平级确认」机制——当机关部门之间互相派发任务时，须双方负责人确认后才正式下发。建议为每个机关部门设置负责人。

⚠ 注意：「部门负责人」与「有 task:reception 权限的成员」是两个不同概念。负责人管「平级确认」，reception 权限管「指派承办人」，可以是同一人，也可以不同。

虚拟机构说明

系统中存在「虚拟机构」（如"公司机关"、"基层单位"），它们是用于归类的壳节点，不是真实的单位，不建议直接派发任务给虚拟机构。真实的部门/单位在其下一级。

👤 用户管理
管理员工账号、组织归属、角色与自定义字段

新建用户

  1. 后台「用户管理」→ 点击右上角「新建用户」
  2. 填写姓名、员工编号（工号，系统唯一标识）、手机号
  3. 选择所属组织（行政机构），可多选
  4. 选择角色（决定该用户能使用哪些功能）
  5. 可在「自定义字段」tab 填入额外信息（入职日期、身份证号等）
💡 员工编号 vs 账号：系统统一使用「员工编号」而非「账号」称呼，登录时也以员工编号识别身份。请勿将它理解成工资系统的工号，它只是本系统的用户标识。

多组织归属

一个用户可以同时归属多个部门（如"综合办公室"兼任"党支部"），系统用「主部门」标记主要归属，影响任务派发时的责任判定。

自定义字段

管理员可在「用户自定义字段」页面定义额外的用户属性（如"入职日期"、"岗位类别"），定义后在用户详情页可填写对应值。字段类型支持文本、日期、数字、下拉选择等。

头像说明

  • 用户可上传本人照片，系统提供 AI 生成职场风 3D 头像（红底、提亮效果）
  • 生成的头像会保存到「头像库」，可随时切换历史头像
  • 头像在首页右上角、后台右上角、桌面客户端均统一显示

🏅 证书管理
覆盖「设计模板 → 颁发证书 → 公开验真」完整闭环

整体流程

1. 设计模板 → 2. AI 提取 / 手动填写 → 3. 审核发证 → 4. 下载 PDF → 5. 扫码验真

① 设计证书模板

后台「证书管理」→「证书模板」→「新建模板」，进入可视化设计器。

  • 8种元素：文本、变量文本、图片、背景、矩形、线条、印章、二维码
  • 变量绑定：拖入「变量文本」元素，点击可选择绑定到「姓名」「荣誉名称」「表彰年度」「证书编号」等动态变量，发证时自动替换
  • 撤销/重做：支持 50 步历史，快捷键 Ctrl+Z / Ctrl+Y
  • 预览：设计界面右上角可随时预览实际效果
  • 导出：模板支持导出为 PNG 和 PDF 两种格式
💡 最佳实践：模板建议以「荣誉类别」为单位创建（如"优秀共产党员证书"）。背景图片建议上传 A4 尺寸高清图片（2480×3508px），避免打印模糊。

② 发证向导（5步）

  1. AI 上传识别：上传表彰文件（Word/PDF），AI 自动提取荣誉名称、年份、受表彰人列表。或直接跳过手动填写
  2. 选择模板：从已有模板中选择，预览效果
  3. 填写受表彰人：输入姓名或工号自动带出，系统会自动填充所属单位全称路径
  4. 确认信息：核对证书编号格式（自动生成：年份-荣誉代码-总数-序号）
  5. 发证：生成 PDF，可单张下载或批量打包 ZIP 下载
⚠ 重名处理：按姓名搜索时如有重名，系统会标注「待核对」，请务必核实工号确认是否为同一人，避免发错证书。

③ AI 自动提取

上传表彰文件后，AI（DeepSeek）会自动识别：

  • 表彰荣誉名称（如"优秀共产党员"）
  • 表彰年度（默认取上一年）
  • 受表彰人名单（姓名+单位）

识别结果为预填值，人工确认后再发证，AI 可能有偏差需校对。

💡 AI 配置：如 AI 提取很慢，可在后台「外部 API 接入」中将 DeepSeek 模型改为 flash 版本（速度更快，精度略低）。

④ 证书验真

每张证书都带有唯一二维码/验真链接，任何人无需登录，直接访问即可验证真伪：

  • 扫描证书上的二维码，或访问 /verify/[token]
  • 显示证书缩略图、颁发信息、颁发时间
  • 已撤销的证书显示「已撤销」标记

⑤ 外部证书导入

对于历史纸质证书或外部系统证书，可通过「外部录入」功能上传原件扫描件，录入基本信息，统一管理。

证书编号规则

2024 - YXGCDYY - 0050 - 0003

表彰年度 - 荣誉代码 - 该荣誉总颁发数 - 本批序号
📋 任务分派系统
通用「下发-填报-审核-汇总」全流程，适用于数据报送、党建报告等场景

角色分工

📤
派发人
有 task:manage 权限，创建任务并派发给各单位/部门
📥
接收管理员
有 task:reception 权限，指派承办人给本部门成员
✍️
填报人
有 task:fill 权限，填写回执并提交

新建任务（4步向导）

  1. AI 上传识别：上传通知文件（Word/PDF），AI 自动生成任务标题、填报要求、截止日期，并建议填报字段和派发范围。文件同时作为通知文件下发给填报人。
    💡 若没有文件，直接跳过此步手动填写也可以。AI 识别出的字段仅供参考，可在第二步修改。
  2. 设计填报表单：使用可视化字段设计器，拖拽添加字段。
    • 支持 9 种字段类型：文本、多行文本、数字、日期、下拉选择、文件、图片、富文本、在线文档
    • 可「按填报要求生成字段」让 AI 自动产出字段方案
    • 可「复制往期任务字段」直接复用历史表单
    • 字段分组：左侧「添加分组」创建分组，字段可跨组拖拽
  3. 选择派发对象：在组织树中勾选目标部门/单位，或直接勾选个人。
    • 「快捷组」：可将常用的派发范围存为自定义快捷组，下次一键选择
    • 「AI 建议范围」：若第一步上传了文件，此处会显示 AI 建议的派发范围，可一键选中
  4. 确认并派发：核查任务信息、字段列表、派发对象数量，点击「确认派发」。
⚠ 权限提示：新建任务须使用有 task:manage 权限的账号（如 admin）。普通员工账号派发会提示 403 权限不足。

平级确认机制

当机关部门向其他机关部门派发任务时，会触发「平级确认」流程：

  • 派发后任务状态为「待确认」，不会立即进入对方待办
  • 双方部门负责人需在「我的待办 → 待我确认」中点击「同意下发」
  • 任一方驳回，该派发对象作废（不影响同任务其他对象）
  • 派发人可对「已驳回」对象点击「重新发起」重新触发确认流程
💡 何时触发：派给「基层单位」或「个人」不触发平级确认，只有「机关↔机关」跨部门才需要确认。

我的待办（填报侧）

有填报任务时，登录后在「我的待办」页面可看到：

  • 待接收：已派发但尚未有人认领的任务，点击「接收」或「指派」
  • 我负责的：已认领/被指派给我的任务，点击「填报」

填报注意事项

  • 可先「保存草稿」，后续再修改提交
  • 提交后不可直接修改，如需更改需等审核人「退回」后重新填报
  • 页面顶部显示派发部门、派发人姓名及联系电话，有疑问可直接拨打
  • 截止日期过后系统不会立即锁定，但超期满 1 个月未提交的回执系统会自动标为通过

超期自动通过规则

ℹ️ 说明：任务截止日期满 1 个月后，仍处于「已提交」待审状态的回执，系统每天凌晨 3 点自动标记为「已通过」，并在审核备注中注明「系统自动通过」。此规则保证历史数据不会永久停留在待审状态。
📖 字典管理
管理系统内下拉选项的可选值

什么是字典

字典是系统中各类「下拉选项」的数据来源，例如「职务」「学历」「荣誉级别」等字段的可选值都从字典中读取。

字典结构（2级）

  • 字典类别（一级）：如"职务"、"学历"，相当于一组下拉的名称
  • 字典项（二级）：如"职务"下有"总经理"、"部门经理"、"员工"等具体选项

操作说明

  1. 后台「字典管理」→ 左侧选择或新建字典类别
  2. 右侧列表管理该类别下的字典项（增删改排序）
  3. 修改立即生效，前台下拉选项同步更新
💡 注意：任务分派中的「下拉选择」字段不依赖字典，而是在字段设计时直接输入选项文本，更灵活。字典主要用于用户管理等固定字段。

🎨 AI 头像生成
上传本人照片，AI 生成职场风 3D 头像

使用步骤

  1. 进入后台「用户管理」→ 找到自己的账号 → 点击「基本信息」tab
  2. 在头像区域点击「上传照片」，选择本人清晰正面照（支持 JPG/PNG）
  3. 点击「AI 生成头像」，等待约 10~30 秒
  4. 生成的头像自动加入「头像库」，可点击选用
  5. 点击保存后，全站头像同步更新
💡 照片建议：使用正面、光线均匀的证件照或职场照，效果最佳。侧脸、戴墨镜、遮挡面部的照片识别效果较差。
ℹ️ 原图保存：上传的原始照片会按「姓名-工号」命名保存到服务器，下次生成头像无需重新上传。
⚠ 费用说明：AI 头像生成调用火山引擎 Seedream 接口，每次生成有 API 费用，请勿频繁生成。
🖥️ 桌面客户端
Windows 桌面应用，后台驻留，新任务自动弹通知

为什么要用桌面客户端

内网 HTTP 环境下，浏览器的推送通知无法工作（PWA 需要 HTTPS）。桌面客户端解决了这个问题：

  • 常驻系统托盘，不占用桌面空间
  • 每 90 秒后台轮询一次待办任务
  • 有新的「待接收」任务时，弹出原生 Windows 通知
  • 点击通知直接打开对应任务

安装与使用

  1. 从管理员处获取安装包（.msi 或 .exe）
  2. 双击安装，可能出现 SmartScreen 提示（点「仍要运行」）
  3. 安装后程序自动启动，右下角任务栏托盘出现党建益友图标

党建益友平台功能建设情况汇报

企业党建数字化应用底座 · 建设阶段总结
汇报时间：2026年6月

一、建设背景与目标

随着企业信息化建设持续推进，各职能部门分散建设了党建、办公、培训、考核等多个业务系统，存在入口分散、数据孤岛、重复建设等问题。为此，启动建设「党建益友」企业级应用底座平台，实现以下目标：

  • 统一接入各内部系统，提供统一门户导航与搜索
  • 构建组织、用户、权限统一管理底座
  • 承载党费缴纳、活动报名、荣誉证书、任务报送等党务业务功能
  • 为后续业务模块快速上线提供技术脚手架

平台定位为「先跑起来，接入真实客户，按需扩展」，避免过度超前建设，以最小可用产品快速交付价值。

二、技术架构

平台采用前后端分离架构，后端为模块化单体，为后续功能扩展预留良好结构：

层次技术选型说明
前端React 19 + Vite + TypeScript + Tailwind CSS组件化、类型安全、极速热更新
后端NestJS + Prisma ORM模块化架构，代码边界清晰，守 DAG 无循环依赖
数据库SQLite（开发）/ PostgreSQL（生产）零配置开发，生产一键切换
文件存储本地盘（Driver 抽象）挂载群晖共享盘即可使用群晖存储，代码无需修改

部署Docker Compose一键部署，支持局域网访问
桌面端Tauri（基于 WebView2）复用 Web 前端，无需二次开发，打包为 .exe 安装包

（一）平台底座

  • 统一门户首页：各系统导航卡片、拼音搜索、访问排行（金/银/铜牌）
  • 站点设置：平台名称、LOGO、主题色（全站实时生效）、页脚配置
  • 组织管理：行政机构四级树（一级单位→四级单位）+ 党组织五类树（党委/总支/支部/临时支部/党小组）
  • 用户管理：多组织归属、多角色授权、自定义扩展字段
  • 角色权限：基于角色的精细化权限控制，权限点精确到具体操作
  • 数据字典：两级字典管理，支撑系统内所有下拉选项
  • 审计日志：全操作留痕，记录操作人、操作内容、时间
  • 统一文件存储：StorageDriver 抽象层，本地盘默认、群晖挂载即用、S3 接口预留

（二）荣誉证书管理系统

覆盖「设计→颁发→验真」完整闭环，核心创新点：

  • 可视化模板设计器：支持8种元素（文本/图片/变量/印章/二维码等）、撤销重做50步、PNG/PDF导出
  • AI 智能提取：上传表彰文件（Word/PDF），AI 自动提取荣誉名称、年份、获奖人名单，大幅降低录入工作量
  • 批量发证：支持单证和批量操作，证书编号自动生成（年份-荣誉代码-总数-序号格式）
  • 公开验真：每张证书唯一二维码，任何人无需登录即可扫码验真
  • PDF 优化：证书 PDF 内图片采用 JPEG 压缩（约 1.5MB/张），支持局域网高效下载
  • 外部证书导入：历史纸质证书扫描件统一归档管理

（三）任务分派系统

通用「下发-填报-审核-汇总」底座，适用于数据报送、党建报告、专项信息收集等场景：

  • 智能创建向导：上传通知文件 → AI 自动解析生成任务结构（标题/要求/截止日期/字段/范围建议）
  • WYSIWYG 字段设计器：9种字段类型（含文件、图片、富文本、在线文档链接），拖拽排序，字段分组，撤销重做
  • 精准派发：组织树多选部门/单位或个人，支持快捷组（常用范围一键复用）
  • 平级确认机制：机关部门间互派须双方负责人确认后才正式下发，规范横向协作流程
  • 承办人指派：接收管理员可将任务指派给本部门具体成员，支持区域权限控制
  • 全程状态追踪：派发→待确认→待接收→填报中→已提交→已审核，状态实时可见
  • 超期自动处理：截止满1个月未审核的回执，系统每日凌晨自动标记为通过，避免数据积压
  • 附件汇总下载：按单位归类打包，文件名规范，一键获取所有附件

（四）AI 能力集成

  • 多 AI 服务商统一接入（DeepSeek / 豆包 / 通义千问），优先级可配置，可热切换
  • 证书 AI 提取：Word/PDF 文件 → 荣誉信息自动抽取
  • 任务 AI 识别：通知文件 → 任务结构自动生成（含填报字段建议）
  • AI 头像生成：员工照片 → 职场风 3D 仿真头像（火山引擎 Seedream 5.0）
  • 3D 模型生成接口预留：火山引擎 Seed3D（为虚拟展厅做准备）
  • 提示词集中管理后台：AI 提示词可后台配置，无需改代码即可调优

（五）多端访问

  • Web 端（浏览器）：前后台一体，局域网 IP 直访
  • Windows 桌面客户端（.exe/.msi）：Tauri 封装，托盘常驻，新任务原生通知弹窗
  • 局域网移动端：手机/平板浏览器访问，响应式自适应布局
  • 公开验真页面：无需登录，二维码/链接直达证书验真

四、建设亮点

1. AI 贯穿全流程，大幅降低人工录入负担

证书颁发和任务派发是频率最高的两项党务工作。传统方式需要人工逐条录入表彰信息、逐份填写任务通知，耗时费力。平台将 AI 能力无缝嵌入：上传一份 Word 文件，几十秒内自动生成完整的证书发放方案或任务派发方案，人工只需确认修改，录入工作量减少 70% 以上。

2. 数据驱动设计，扩展成本极低

平台核心模块（任务字段、用户扩展字段、导航项、证书变量）均采用元数据驱动设计，新增一种业务需求无需修改代码，只需在后台配置即可生效。这种设计使平台能够快速响应各部门不同的业务场景，而不是定制开发。

3. 模块化架构，业务边界清晰

后端采用严格的模块化单体架构，代码层面强制 5 条约束（表归属标注、跨模块 DI 调用、barrel 入口、依赖图无环、固定骨架）。新增业务模块按 7 步清单操作，任何开发者接手均可快速上手，不会产生"改了这里，不知道影响了哪里"的混乱。

4. 本地化部署，数据安全可控

平台完全支持内网离线部署，文件存储对接企业现有群晖 NAS（挂载共享盘即用），AI 模型调用仅通过 API 发送文本内容（不上传原始文件），核心数据全程不离开企业内网。

5. 平级确认机制，规范横向协作

机关部门间互发任务须双方负责人确认，避免了"被动接单"和"无效派发"的问题，同时保留派发人的"重新发起"权限和管理员的"代确认"权限，在规范流程的同时防止流程死锁。

六、总结

党建益友平台以「够用就好、快速迭代」为原则，在较短周期内完成了底座建设和多项核心业务功能落地。平台技术架构具备良好的扩展性，后续每新增一个业务模块，只需在现有框架内填充，不需要推倒重来。

当前建设重点是把已有功能打磨到可交付给真实用户使用的质量水平，同时推进统一身份认证接入，为正式推广上线做准备。
